👤 Analista de Validação
📋 Pipeline: Validação Comercial
Ação Manual · Estágio: Para Validação (1288618541)
O analista atribuído revisa os dados, documentos e pendências do negócio
✋ Manual
Responsável
Analista de Validação Comercial atribuído (Joao Vitor ou Leticia Ananda)
O que o analista verifica
1
Documentação do cliente (fatura, identidade, contrato)
2
Dados preenchidos no card (e-mail, telefone, CPF/CNPJ, consumo)
3
Consistência das informações do negócio
4
Elegibilidade do cliente (inadimplência, benefício tarifário, fraude)
Decisão Manual · Propriedades de Controle
Como o analista sinaliza o resultado da análise
✋ Manual
Propriedade principal
existe_pendencia_para_validacao_comercial — é o coração do sistema. Define o que acontece a seguir:

→ false = aprovado sem pendências → aciona a Fase 3A (Fechamento)
→ true = há problemas → aciona a Fase 3B (Pendências/Cancelamento)
→ vazio = aguardando revalidação após correção → aciona a Fase 3C
Propriedade secundária (obrigatória se true)
pendencia_comercial_validacao — deve ser preenchida com um valor da lista de pendências cadastradas. Valores não reconhecidos deixam o deal parado sem ação.
🚨
Ponto crítico do sistema: Se o analista não preencher existe_pendencia_para_validacao_comercial após a análise, nenhum dos próximos workflows é acionado e o negócio fica parado em "Para Validação" sem alerta para ninguém.
Propriedades-chave: existe_pendencia_para_validacao_comercial pendencia_comercial_validacao
