Fluxos n8n
Detalhes dos fluxos n8n integrados ao ciclo operacional.
n8n Criação de Tickets (JSON fornecido anteriormente)

Este fluxo n8n é responsável por agrupar negócios por usina/gerador/alocação e criar tickets de rateio no HubSpot, associando os negócios correspondentes. Ele também atualiza a propriedade `ticket_de_cobranca` para `true` nos negócios após a criação do ticket.

Principais Ações:

  • Busca deals no HubSpot com `dealstage` "1119360529" e `ticket_de_cobranca` = false.
  • Agrupa deals por `id_usina_alocacao_e_rateio`, `id_gerador_alocacao_e_rateio` e `alocacao_ou_substituicao_fluxo_de_alocacao_`.
  • Calcula o `consumo_total` dos negócios agrupados.
  • Cria um novo ticket no HubSpot com informações da usina, gerador, tipo de rateio e associa os deals correspondentes.
  • Atualiza os deals associados, definindo `ticket_de_cobranca` como `true`.
n8n Merge de Tickets (JSON fornecido anteriormente)

Este fluxo n8n gerencia a fusão de tickets no HubSpot e a organização de arquivos no Google Drive.

Principais Ações:

  • Agrega tickets por usina/gerador para identificar um ticket principal e tickets secundários a serem mergeados.
  • Captura associações de deals para os tickets.
  • Realiza o merge de tickets secundários no ticket principal no HubSpot.
  • Move arquivos do Google Drive dos tickets secundários para a pasta do ticket principal.
  • Atualiza o `soma_do_consumo_dos_negocios_associados_kwh` no ticket principal.
n8n Geração de Contratos (Integrado ao WF Ticket 1)

Acionado pelo WF Ticket 1, este fluxo automatiza a geração dos documentos de contrato.

Principais Ações:

  • Recebe dados do ticket e dos negócios associados.
  • Utiliza templates para gerar os contratos (ex: PDF, DOCX).
  • Armazena os contratos gerados em um local temporário ou no Google Drive.
  • Atualiza a propriedade `contratos_do_ticket_gerados` no HubSpot.
n8n Envio para ClickSign (Integrado ao WF Ticket 2)

Acionado pelo WF Ticket 2, este fluxo envia os contratos gerados para a plataforma ClickSign para coleta de assinaturas.

Principais Ações:

  • Pega os contratos gerados pelo fluxo anterior.
  • Cria um novo envelope de assinaturas no ClickSign.
  • Adiciona os signatários e os documentos ao envelope.
  • Envia o envelope para assinatura.
  • Atualiza a propriedade `contratos_do_ticket_no_clicksign` no HubSpot.
n8n Checagem de Assinaturas (Integrado ao WF Ticket 3)

Acionado pelo WF Ticket 3, este fluxo monitora o status das assinaturas na ClickSign.

Principais Ações:

  • Consulta o status do envelope de assinaturas na ClickSign.
  • Verifica se todos os documentos foram assinados.
  • Atualiza a propriedade `contratos_de_negocios_do_ticket_assinados` no HubSpot para `True` (com 'T' maiúsculo, pois é case-sensitive).
n8n Baixar Documentos (Integrado ao WF Ticket 4)

Acionado pelo WF Ticket 4, este fluxo baixa os documentos assinados e os organiza no Google Drive.

Principais Ações:

  • Baixa os documentos finais (contratos assinados) da ClickSign.
  • Cria uma pasta específica no Google Drive para o ticket (se ainda não existir).
  • Faz o upload dos documentos baixados para essa pasta.
  • Atualiza as propriedades `link_anexos_drive` e `documentos_baixados` no HubSpot para `True` (case-sensitive).
n8n Cadastro no OPS (Integrado ao WF Ticket 5)

Acionado pelo WF Ticket 5, este fluxo realiza o cadastro dos negócios no sistema OPS.

Principais Ações:

  • Pega os dados necessários do ticket e dos negócios associados.
  • Integra com a API do sistema OPS para cadastrar os negócios.
  • Verifica o sucesso do cadastro.
  • Atualiza a propriedade `negocios_do_ticket_cadastrados_no_ops` no HubSpot para `true`.
Propriedades HubSpot Chave
Propriedades essenciais para o funcionamento dos workflows e fluxos n8n.
Propriedade Tipo Descrição
`dealstage` Estágio do Negócio Etapa atual do negócio no pipeline. Gatilho para diversos WFs.
`ticket_de_cobranca` Booleano Indica se um ticket de cobrança já foi criado para o deal. Usado no n8n de criação de tickets.
`existe_pendencia_para_validacao_comercial` Lista suspensa Indica se há pendências na validação comercial. Usado no WF1 e WF4.
`troca_de_titularidade` Booleano Indica se o negócio envolve troca de titularidade. Usado no WF Bastão Comercial ↔ CS.
`nome_da_usina` Texto Nome da usina associada ao deal. Usado no n8n de criação de tickets.
`gerador_alocacao` Texto Nome do gerador associado ao deal. Usado no n8n de criação de tickets.
`id_usina_alocacao_e_rateio` Texto ID da usina para alocação e rateio. Usado no n8n de criação de tickets.
`id_gerador_alocacao_e_rateio` Texto ID do gerador para alocação e rateio. Usado no n8n de criação de tickets.
`alocacao_ou_substituicao_fluxo_de_alocacao_` Lista suspensa Tipo de alocação (nova, substituição). Usado no n8n de criação de tickets.
`consumo_medio_em_kwh` Número Consumo médio em kWh do beneficiário. Usado no n8n de criação de tickets.
`soma_do_consumo_dos_negocios_associados_kwh` Número Soma do consumo dos deals associados ao ticket. Atualizado pelo n8n de merge.
`contratos_do_ticket_gerados` Booleano Indica se os contratos do ticket foram gerados. Atualizado pelo n8n.
`contratos_do_ticket_no_clicksign` Booleano Indica se os contratos foram enviados para ClickSign. Atualizado pelo n8n.
`contratos_de_negocios_do_ticket_assinados` Booleano Indica se todos os contratos foram assinados. Atualizado pelo n8n (True 🔴).
`link_anexos_drive` URL Link da pasta no Google Drive com os documentos. Atualizado pelo n8n.
`documentos_baixados` Booleano Indica se os documentos foram baixados e organizados. Atualizado pelo n8n (True 🔴).
`negocios_do_ticket_cadastrados_no_ops` Booleano Indica se os negócios do ticket foram cadastrados no OPS. Atualizado pelo n8n.
`protocolo_de_rateio` Texto Protocolo de rateio. Uma das 10 condições para o WF7.
Troubleshooting e Pontos de Atenção
Problemas comuns e como identificá-los no fluxo.
⚠️
ATENÇÃO · Owner do Deal
Se o owner do deal for alterado manualmente após a atribuição inicial, o owner anterior permanece atribuído se não for removido manualmente.
⚠️
ATENÇÃO · Branch AlugSol duplicado no WF1
O path de Gabriela Bruno está inacessível. Todos os deals AlugSol vão para Joao Vitor / Leticia Ananda.
⚠️
ATENÇÃO · Campos de alocação incompletos
Se qualquer um dos 4 campos (usina, gerador, IDs) estiver vazio, WF Alocação não avança para Cobrança. Sem alerta.
⚠️
ATENÇÃO · Ticket sem vínculo com o negócio
WF Ticket 1 e WF7 dependem da associação ticket → deal. Sem ela, a ação de mover dealstage é ignorada silenciosamente.
